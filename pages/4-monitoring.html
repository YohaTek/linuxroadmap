4. System / Server Review & Monitoring

Diagnosing system health, resource usage, and logs.

1. Concept Overview

System monitoring is about answering three questions: "Is it up?", "Is it slow?", and "Why?".

Linux provides a wealth of metrics exposed through the /proc filesystem and various tools. You need to monitor CPU (processing power), Memory (RAM), Disk I/O (read/write speed), and Network (bandwidth/latency).

2. Key Commands

4. Deep Dive: Load Average

  

    Load average is NOT just CPU usage. It is the number of processes that are either using the CPU OR
    waiting for disk I/O.
  

  

    A load of 1.00 on a single-core system means the CPU is 100% busy. On a quad-core system, 1.00 means 25% busy.
    
High load + Low CPU usage usually means Disk I/O bottleneck.
  






  
5. Common Pitfalls

  
    
    
  • Misinterpreting "Free" RAM: Linux uses unused RAM to cache files. free might show
      very little "free" memory, but if "available" is high, you are fine.
    • 
    
  • Tail -f on huge files: Tailing a multi-gigabyte log file can slow down your terminal. Use
      tail -n 100 first.
    •
