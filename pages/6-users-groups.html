6. User & Group Management

Managing identities, passwords, and access control.

1. Concept Overview

Linux identities are defined by files, not a database.

  • /etc/passwd: Stores user info (Username, UID, Home Dir, Shell).
  • /etc/shadow: Stores encrypted passwords (secure).
  • /etc/group: Stores group memberships.

UID (User ID) is the number the kernel cares about. Root is always UID 0. Regular users usually start at 1000.

2. Key Commands

4. Deep Dive: /etc/passwd

  

    A line in /etc/passwd looks like this:
    
root:x:0:0:root:/root:/bin/bash
  

  
    
    
  1. root: Username
    2. 
    
  2. x: Password placeholder (real hash is in /etc/shadow)
    3. 
    
  3. 0: User ID (UID)
    4. 
    
  4. 0: Group ID (GID)
    5. 
    
  5. root: Comment (GECOS)
    6. 
    
  6. /root: Home Directory
    7. 
    
  7. /bin/bash: Default Shell
    8. 
  






  
5. Common Pitfalls

  
    
    
  • Forgetting -m: useradd bob (without -m) creates a user
      without a home directory. Bob will be homeless and sad.
    • 
    
  • Editing /etc/passwd manually: Dangerous! A typo can lock everyone out. Use vipw if
      you must, but stick to usermod commands.
    •
