13. Containerization & Related Topics

Docker, isolation, and shipping applications.

1. Concept Overview

Containers allow you to package an application with all its libraries and dependencies into a single "Image".

Unlike Virtual Machines (which simulate a whole computer), Containers share the host Linux kernel but are isolated using Namespaces (visibility) and Cgroups (resource limits). This makes them lightweight and fast.

4. Deep Dive: Layers

  

    Docker images are built in Layers. Each line in a Dockerfile creates a new layer.
  

  

    Layers are cached. If you change the last line of your Dockerfile, Docker only rebuilds that last layer, not the
    whole thing. This makes building images very fast.
  






  
5. Common Pitfalls

  
    
    
  • Data Persistence: Files created inside a container disappear when you delete the container. Use
      Volumes (-v /host/path:/container/path) to save data permanently.
    • 
    
  • "It works on my machine": It usually does with Docker, but check your architecture! (ARM vs
      x86).
