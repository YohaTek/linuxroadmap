9. Disks & Filesystems

Partitions, mounting, and monitoring disk usage.

1. Concept Overview

In Linux, hardware devices are files. Your hard drive is likely /dev/sda or /dev/nvme0n1.

You don't access the raw disk. You access a Partition (e.g., /dev/sda1) formatted with a Filesystem (like ext4 or xfs). This filesystem is then Mounted to a directory (like /home or /mnt/usb).

4. Deep Dive: Inodes

  

    Sometimes df -h says you have space, but you can't create files. You might be out of
    Inodes.
  

  

    Every file takes up 1 Inode (index node) to store metadata. If you have millions of tiny files, you can run out of
    inodes before you run out of gigabytes.
    
Check with: df -i.
  






  
5. Common Pitfalls

  
    
    
  • Unmounting busy drives: You can't umount a drive if you are currently
      cd'd inside it. You'll get "target is busy". cd out first!
    • 
    
  • fstab errors: If you mess up /etc/fstab, your system might fail to boot. Always
      test with sudo mount -a before rebooting.
    •
