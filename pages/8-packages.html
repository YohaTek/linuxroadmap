8. Package Management

Installing, updating, and removing software.

1. Concept Overview

Unlike Windows (downloading .exe files from websites) or macOS (dragging .app to Applications), Linux uses Package Managers.

Software is stored in centralized Repositories (repos). The package manager handles downloading the software AND all its dependencies (libraries it needs to run).

2. Key Commands

Commands vary by distribution. The two big families are Debian/Ubuntu (apt) and RedHat/Fedora (dnf).

4. Deep Dive: Repositories

  

    Your computer doesn't know about every software in the world. It only knows what is in the lists inside
    /etc/apt/sources.list (or /etc/yum.repos.d/).
  

  

    To install 3rd party software (like VS Code or Chrome), you often need to add their repository URL and GPG key to
    your system first.
  






  
5. Common Pitfalls

  
    
    
  • Partial Upgrades: Don't just run apt upgrade without running
      apt update first. You need the latest list!
    • 
    
  • "Held" packages: Sometimes updates are blocked because of dependency conflicts. Read the error
      message carefully.
    •
