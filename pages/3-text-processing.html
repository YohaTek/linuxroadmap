3. Text Processing & Streams

The power of pipes, grep, and the UNIX philosophy.

1. Concept Overview

The UNIX philosophy states: "Write programs that do one thing and do it well. Write programs to work together. Write programs to handle text streams, because that is a universal interface."

Pipes (`|`) allow you to chain these small tools together. The output (stdout) of one command becomes the input (stdin) of the next. This lets you build powerful data processing pipelines on the fly without writing a single script.

4. I/O Redirection & Standard Streams

  

    In Linux, every process has 3 special data streams connected to it by default. Mastering how to move data between these streams is the key to command-line efficiency.
  

  
    
    
  • stdin (0): Standard Input (data flowing INTO a command, usually from keyboard or pipe).
    • 
    
  • stdout (1): Standard Output (data flowing OUT of a command, usually to the screen).
    • 
    
  • stderr (2): Standard Error (error messages, separate from stdout).
    • 
  


  
Common Redirection Operators

  

    

      > (Overwrite)
      
Redirect stdout to a file (creates if new, overwrites if exists).

      

    

    

      >> (Append)
      
Redirect stdout to a file without erasing current content.

      

    

    

      < (Input)
      
Feed a file's content INTO a command as stdin.

      

    

    

      2> (Error)
      
Redirect ONLY error messages to a separate file.

      

    

    

      &> (Both)
      
Redirect both stdout and stderr to the same destination.

      

    

    

      | (Pipe)
      
The most powerful tool: Connect stdout of Command A to stdin of Command B.

6. Common Pitfalls

  
    
    
  • Useless use of cat: cat file | grep pattern is common but inefficient. Use
      grep pattern file instead.
    • 
    
  • Redirecting to input file: sort file.txt > file.txt will erase
      the file before reading it! Use a temporary file or sort -o file.txt file.txt.
    •
