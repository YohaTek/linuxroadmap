1. Navigation Basics

Mastering the filesystem tree, paths, and essential movement commands.

1. Concept Overview

In Linux, everything starts from the root directory, represented by a single forward slash /. Unlike Windows, which has drive letters (C:, D:), Linux has a single unified directory tree. Drives and devices are "mounted" onto directories within this tree.

Navigation is the art of moving through this tree efficiently. It relies on understanding Absolute Paths (full address from root, e.g., /home/user/docs) vs. Relative Paths (directions from where you are, e.g., ../docs).

4. Deep Dive: The "." and ".."

  

    Every directory contains at least two special entries:
    
    
      
  • . (dot): Represents the current directory. Useful for executing scripts in the current folder (./script.sh).
    • 
      
  • .. (dot-dot): Represents the parent directory. Used for relative navigation.
    • 
    

    Try ls -la to see them explicitly!
  






  
5. Common Pitfalls

  
    
    
  • Case Sensitivity: File.txt and file.txt are different files.
    • 
    
  • Spaces in Names: Avoid them. If you must, use quotes: cd "My Documents" or escape them: cd My\ Documents.
    • 
    
  • Root Directory vs. Home Directory: / is the system root. ~ (or /home/username) is your personal home. Don't mix them up!
    •
