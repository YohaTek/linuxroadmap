7. Service / Daemon Management

Managing background services with systemd.

1. Concept Overview

A Service (or Daemon) is a program that runs in the background, waiting to handle requests (like a web server) or performing scheduled tasks.

Most modern Linux distributions use systemd as the init system (PID 1). It starts everything else. You control it with the systemctl command.

2. Key Commands

systemctl (Control)

Manage service state.

4. Deep Dive: Targets

  

    Systemd uses "Targets" instead of old-school Runlevels.
  

  
    
    
  • multi-user.target: Normal state (Text mode, Networking enabled). Equivalent to Runlevel 3.
    • 
    
  • graphical.target: GUI mode. Equivalent to Runlevel 5.
    • 
    
  • rescue.target: Single user mode for repairs.
    • 
  






  
5. Common Pitfalls

  
    
    
  • Editing unit files without reloading: If you change a .service file, you MUST run
      sudo systemctl daemon-reload or systemd won't see the changes.
    • 
    
  • Enable vs Start: enable only configures the next boot. start
      runs it now. Use enable --now to do both.
    •
