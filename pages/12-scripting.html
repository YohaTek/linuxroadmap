12. Shell Programming / Scripting

Automate tasks with Bash scripts.

1. Concept Overview

A shell script is just a text file containing a list of commands you would run manually.

It starts with a Shebang (#!/bin/bash) which tells the system which interpreter to use. Scripts allow you to use variables, loops, and conditionals to build complex automation logic.

4. Deep Dive: Exit Codes

  

    Every command returns a number when it finishes: Exit Code.
    
0 = Success.
    
1-255 = Error.
  

  

    You can check the last exit code with $?.
    
ls /fakefile; echo $? will print 2 (No such file).
  






  
5. Functions & Reuse

  

    Functions let you group logic and call it multiple times, keeping scripts readable and DRY (Don’t Repeat Yourself).
  

  






  
6. Handling Arguments Safely

  

    For small scripts, positional parameters are enough. For larger ones, parsing flags improves UX.
  

  


  






  
7. Loops, Globs & Text Processing

  
Shell scripts shine at iterating over files and piping through Unix tools like grep, sed, and awk.

  


  


  






  
8. Safety Patterns

  

    Small safety habits prevent destructive mistakes in production scripts.
  

  

  
    
    
  • set -euo pipefail: Fail fast on errors, undefined variables, and pipe failures.
    • 
    
  • IFS: Narrowed to newline/tab to avoid splitting on spaces inside filenames.
    • 
    
  • Defensive checks: Validate arguments and paths before rm or mv.
    •

12. Common Pitfalls

  
    
    
  • Whitespace matters: if [ $a == $b ] needs spaces inside the brackets.
      if [$a==$b] will fail.
    • 
    
  • Unquoted variables: Always quote your variables ("$NAME") to handle filenames with
      spaces correctly.
