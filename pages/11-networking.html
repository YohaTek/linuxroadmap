11. Networking

IPs, DNS, Ports, and troubleshooting connectivity.

1. Concept Overview

Linux networking is built on the TCP/IP stack. Every interface (like eth0 or wlan0) has an IP address.

To talk to other computers, you need:
1. IP Address: Your identity.
2. Netmask: Defines your local network size.
3. Gateway: The router that connects you to the internet.
4. DNS: Translates "google.com" to an IP.

4. Deep Dive: DNS Resolution

  

    When you ping google.com, Linux checks:
    
1. /etc/hosts: Local overrides (static mapping).
    
2. /etc/resolv.conf: Lists your DNS servers (nameservers).
  

  

    Use dig google.com to debug DNS queries specifically.

10. Packet Analysis & Sniffing

  
To diagnose deep network issues, you need to see the actual traffic on the wire using tools like tcpdump or Wireshark.

  






  
11. Firewalls and Security

  
Linux uses Netfilter in the kernel. You can manage it via iptables, nftables, or user-friendly wrappers like ufw.

  






  
12. VPNs and Tunneling

  
VPNs securely bridge networks over the internet. WireGuard is the modern standard, baked into the Linux kernel.

  






  
13. Dynamic Routing

  
Unlike static routes, protocols like OSPF (internal) and BGP (internet) automatically discover paths. FRRouting (FRR) is the leading Linux routing suite.

  






  
14. Network Namespaces

  
This is how Docker isolates containers. A namespace is a completely separate network stack (interfaces, routing tables) on the same machine.

  






  
15. Load Balancing & High Availability

  
Distribute traffic across servers using HAProxy/Nginx (Load Balancing) and ensure uptime with Keepalived (VRRP VIP failover).

  






  
16. Common Pitfalls

  
    
    
  • Firewalls: If you can't connect to a server, check if the firewall is blocking it
      (sudo ufw status or iptables -L).
    • 
    
  • ifconfig vs ip: ifconfig is deprecated. Learn ip commands.
    • 
    
  • Asymmetric routing: Packets go out one path and return another. Use traceroute from both
      directions and verify routes on every router.
    • 
    
  • Firewall between segments: Each Linux router may have iptables or nftables rules
      blocking forwarding. Temporarily disable or add explicit ACCEPT rules on FORWARD.
    • 
    
  • Wrong VLAN tagging: If one side of a link is trunk/tagged and the other is access/untagged, traffic
      will disappear. Always check switch port mode vs Linux VLAN config.
    • 
    
  • Duplicate subnets: Using the same subnet on multiple interfaces/segments is a classic source of
      confusion. Ensure each link/LAN has a unique network.
    •
