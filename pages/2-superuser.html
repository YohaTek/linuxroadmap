2. Super User & File Operations

Understanding permissions, ownership, and the power of sudo.

1. Concept Overview

Linux is a multi-user system by design. To protect the system from accidental (or malicious) damage, regular users have limited access. They can only modify files in their own home directory.

The Super User (often named root) has unrestricted access to the entire system. To perform administrative tasks (like installing software or editing system configs), you must temporarily elevate your privileges using sudo (SuperUser DO).

2. Key Commands

4. Deep Dive: Octal Permissions

  

    Permissions are often represented by 3 numbers (e.g., 755). Each number represents the sum of
    permissions for User, Group, and Others.
  

  
    
    
  • Read (r) = 4
    • 
    
  • Write (w) = 2
    • 
    
  • Execute (x) = 1
    • 
  

  

    So, 7 = 4+2+1 (Read+Write+Execute). 5 = 4+1 (Read+Execute).

    chmod 755 means: User gets everything (7), Group gets Read+Exec (5), Others get Read+Exec (5).
  






  
5. Common Pitfalls

  
    
    
  • Running everything as root: Don't log in as root or use sudo for everything. It's
      a security risk.
    • 
    
  • chmod 777: Never set permissions to 777 (everyone can write) unless you absolutely know what
      you are doing. It's a major security hole.
    • 
    
  • Recursive chown/chmod: Be very careful with -R. Running
      sudo chown -R user / can destroy your system.
    •
