10. Boot & Logs

Understanding how Linux starts and how to find errors.

1. Concept Overview

The Linux boot process is a chain reaction:
BIOS/UEFI -> Bootloader (GRUB) -> Kernel -> Init System (systemd) -> User Space.

If something fails, the system logs it. Modern Linux uses journald (binary logs) and syslog (text logs) to record events.

4. Deep Dive: The Init Process

  

    PID 1 is the first process started by the kernel. It is the ancestor of ALL other processes.
  

  

    If PID 1 dies, the kernel panics and the system halts. In modern Linux, PID 1 is systemd. In older
    systems (and containers), it might be init or a shell script.
  






  
5. Common Pitfalls

  
    
    
  • Deleting logs to save space: Don't manually delete /var/log files. Use
      journalctl --vacuum-time=2d to clean up safely.
    • 
    
  • Ignoring "Emergency Mode": If Linux drops you into a root shell at boot, it usually means a
      disk mount failed (check /etc/fstab).
    •
